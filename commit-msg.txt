fix: critical auth bugs causing login-to-401 freeze loop

Three root causes:
1. Middleware called async validateSession() without await — Promise always
   truthy, but route-level requireAuth() DID check. Net: pass middleware, fail
   at route, data load fails, re-renders, infinite loop.
2. HMAC signature corrupted: String.fromCharCode(...spread Uint8Array) can
   hit stack limits or silently corrupt bytes. Signature in cookie never
   matched verifyToken output. Switched to hex encoding.
3. doLoadAll not wrapped in useCallback — recreated every render. When API
   calls fail, loaded stays false, effect re-triggers with new identity,
   causing browser-freezing infinite render loop. Added useCallback +
   loadedRef guard.
