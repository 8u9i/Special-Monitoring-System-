Section-by-Section

🎨 Color System — 8/10

The amber/saffron primary (#D4A23A) on warm cream canvas is a distinctive, intentional choice that avoids the generic blue-purple SaaS reflex. Semantic roles are well-assigned, tag colors have proper text variants, and dark mode is a complete token swap — not a lazy inversion.

  • Consider: Push amber-secondary slightly toward yellow-gold to distinguish it from amber-primary
  • Consider: Deepen canvas from #F5F0EB to #F0EAE4 for 5% surface distinction

🔤 Typography — 7/10

Tajawal + Amiri + Cause + Inter is a thoughtful domain-appropriate pairing. clamp() on headings is a good responsive choice.

  • Fix: --text-2xs at 0.6875rem is below the accessibility floor — bump to 0.75rem
  • Consider: Differentiate prose line-height (1.8) from UI label line-height (1.4-1.5) for scanning efficiency
  • Consider: Use font-weight 800 for the largest heading level for clearer hierarchy

📐 Layout — 7/10

Composition correctly identifies the dominant work pattern as Monitor. Sidebar + header shell is predictable and efficient. The three learning trails share a consistent accordion + student card pattern.

  • Consider: Make the top-student card on the dashboard span 2 columns — it's the most emotionally meaningful metric
  • Consider: Give each learning trail a distinctive entry element (progress ring for quran, daily hadith pull for hadith)

✨ Motion — 4/10 ⚠️

Weakest category. prefers-reduced-motion is respected and buttons have tactile press feedback, but the motion system is nearly absent.

  • Add: Accordion expand/collapse animation — 200ms ease-out height transition
  • Add: Page content fade-in on route change — 200ms, 50ms delay
  • Add: List entrance stagger — 20ms per-item delay on first render

🖱️ Interaction — 6/10

Hover, focus, active, disabled, loading, and empty states are all handled. Escape key and backdrop-click dismiss modals.

  • Fix: Password toggle shows the same lock icon for both states — use visibility/visibility_off
  • Add: Skeleton/placeholder states during loading
  • Add: Brief pulse animation on status toggle confirmation (300ms scale)

📱 Responsive — 7/10

Grid breakpoints well-chosen, mobile sidebar with RTL-aware slide-in, touch targets ≥44px, viewport-fit=cover set.

  • Add: Scroll-snapping to student carousel for mobile polish
  • Fix: Login input font-size (14px) triggers iOS zoom — bump to 1rem on mobile

♿ Accessibility — 6/10

RTL layout, focus-visible outlines, aria-expanded, aria-modal, and Escape handlers are all present.

  • Add: aria-current="page" to active sidebar navigation items
  • Add: role="listbox" + aria-selected to student carousel
  • Add: Skip-to-content link
  • Verify: Tag amber text contrast at 0.6875rem size

🧩 Surface Quality — 7/10

Well-structured component hierarchy: panel, button, tag, badge, progress, empty-state, modal, input — all using design tokens, no hardcoded colors.

  • Consider: A 2px border-radius on buttons and inputs to soften the zero-radius aesthetic
  • Consider: Abstract the hadith/quran accordion into a reusable <AccordionList>

🎙️ Voice / Identity — 8/10

Strong authentic identity. "يا إخوتي" carries warmth, the leaf/sprout icon symbolizes growth, the بسملة grounds the app in its context, educational nudges are phrased warmly.

  • Consider: Remove "v1.0 — متابع الحفظ" from sidebar footer — feels like dev cruft

🌙 Dark Mode — 8/10

Complete token swap, prefers-color-scheme + manual override, dark canvas is genuinely dark, text contrast well-considered. No issues.
